HTML (Hypertext Markup Language) is a markup language (as opposed to a programming langauge). Its purpose is to design documents that will be rendered by a web browser as a web page, like this one! Every component on this page (the navigation bars, the header text, the GIF to the right, this text) is all contained within various tags. Consider the HTML code below: 

      
        <p>This is <b>SO COOL!<b></p>
        <a href="https://www.youtube.com/watch?v=dQw4w9WgXcQ">This is a link</a>

There are 3 tags inside of that snippet:

  • The p tag represents a paragraph. Text is usually stored inside of these.
  • The b tag means that whatever text is inside of this tag will be bolded.
  • The a tag is a link to another web page (or another location within this page. This tag also has an attribute called href. Each tag can have a bunch of attributes. In this case, the value of the href attribute indicates where the link should go to when clicked.
The list above was written inside of an ul tag (short for unordered list). Each element of that list was written inside of an li tag (short for list element).

Circling back: suppose we actually wrote the html code snippet above into a web page. We would see the two lines directly below:

This is SO COOL!

This is a link

Now writing an entire web page using just HTML would be tedious and ugly. HTML is often paired with 2 other languages: CSS and JS.

CSS (Cascading Style Sheets) is used to describe what HTML elements should look like. CSS stylesheets are usually imported into an HTML document at the top. They contain specifications for how specific tags, classes, or IDs should be stylized. Below are two snippets of CSS code. The first specifies that all p tags should use the font Lucida Console, and the second specifies all tags with a class attribute of "swag" should have white text and a blue background. 

      
        p {
          font-family: "Lucida Console", "Courier New", monospace;
        }

        .swag {
          background-color: blue;
          color: white;
        }

In fact, in order to display the CSS code shown above, an html code tag was used. This code tag had been stylized by CSS so that it would have a background color of #f5f5f5 (very light grey), text font of courier, and several other properties.

CSS can control basically anything about how your page elements should look, including:

  • Font family, weight, size, color
  • Element width, height, text alignment, margins, padding
  • Borders, background colors/images
  • What bullet points should lists use
  • How should elements move/resize when the page resizes
  • How elements appearences should change when hoverd/clicked on
  • Table borders, cell dimensions, cell color
  • A bunch of even more interesting things
It is common practice for websites to either start out as a downloaded HTML/CSS template with the styling pre-written, or to import CSS from a library online, or both. In fact, this page was written with a template that imported TONS of CSS. If a lines 8-28 of this HTML file are commented out, you can see what this page looks like without the CSS.

JS (short for JavaScript) is a language much closer to Python. Some of its syntax looks C/C++-esque, but most of its features are extremely similar to Python. Python is better for writing large programs, but JS is better for web development because it is so easily integrated with HTML. The way this is often phrased in industry is that Python is often used for back-end development (everything that users don't see) and Javascript is often used for front-end development (everything that they do see).
