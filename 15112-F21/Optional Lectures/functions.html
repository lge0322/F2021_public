Functional Weirdness

A brief introduction to a couple of functional programming tricks that JS has strong support for. These exist in many languages (including Python), and are often very convenient ways to do some complex calculations.

Arrow Functions

An arrow function (more commonly known in other languages as an anonymous function, a lambda function, or a lambda) is an fucntion created with a single expression in the formats below: 

            
// Takes in 2 variables a and b and adds them
var f1 = (a, b) => a + b;

// Takes in 2 variables a and b and returns the max
var f2 = (a, b) => {
    if (a >= b) {return a;}
    return b;
};

// Takes in a variable a and concatenates it to itself
var f3 = (a) => Number(a.toString() + a.toString());

// Takes in a variable a and returns True if it is even
var f4 = (a) => a % 2 == 0;

If you click the button above and provide 2 numbers as inputs, then JS will call these 4 arrow functions for you and display the results below.

The perk of using arrow functions is that you don't need to even define a function, or bind them to a variable name. You can just create them and immediatly use them for some purpose, as shown below, which creates a quadratic formula function and immediatly calls it on inputs 1, -2, -8, without ever naming the function. 

            
var x = ((a, b, c) => (-b + Math.sqrt(b*b - 4*a*c)) / (2 * a))(1, -2, -8)

Higher Order Functions

Higher Order Functions (HOFs for short) are functions that take in functions as inputs, return functions as outputs, or both. There are many such funcitons, but we'll mention a few extremely common ones that are built into JS. The first is map, which applies a function to an entire list (i.e. squares the elements). The second is filter, which removes all the elements that do not meet a certain criteria (i.e. filters out all perfect squares). The third is reduce, which uses a function to combine all of the elements of a list (i.e. get the sum of a list). These are often used with arrow functions.

If you click the button below, a new list of random 2 digit integers L will be created (using a HOF that is very similar to list comprehension). It is then sorted by the 10s digit of each number (yes, this version of sorting is also a HOF). Then, a new list M is created by filtering L to only keep the odd elements. Then, a new list N is created by mapping the function which takes each element modulo 10 to M. Finally, a value O is obtained by reducing M with a funciton that takes in 2 integers, converts them to strings, and adds them together.
